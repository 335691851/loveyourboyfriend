# Immersive Chat MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a production-ready mobile H5 where an adult anonymous user can exchange persistent text and voice messages with a fictional, emotionally attentive male companion.

**Architecture:** The Next.js client owns anonymous Supabase authentication, optimistic chat state, recording and playback. FastAPI validates the Supabase bearer token, persists through owner-scoped Supabase REST calls, runs a LangChain prompt/streaming chain, extracts typed memories in a second structured-output call, and proxies SiliconFlow transcription and speech generation through its OpenAI-compatible API. The first voice release uses upload/transcribe/chat/TTS for reliability.

**Tech Stack:** Next.js 16, React 19, TypeScript, Supabase JS, FastAPI, LangChain 1.x, langchain-openai, OpenAI-compatible Python SDK, SiliconFlow, Supabase Python SDK, pytest, Vitest.

## Global Constraints

- Audience is adults aged 20–40; the UI clearly identifies the character as fictional and AI-generated.
- Retention comes from continuity, warmth, memory and responsiveness, never guilt, exclusivity pressure or manipulative notifications.
- Anonymous UUID is the sole MVP identity; losing browser storage means losing account recovery.
- Raw messages, audio references and derived memories expire after 90 days.
- Browser receives only the Supabase URL, publishable key and Render API URL.
- All user data access is authorized by a verified Supabase bearer token and owner-scoped RLS.
- Text responses stream immediately; voice uses transcription followed by the same chat chain and TTS.
